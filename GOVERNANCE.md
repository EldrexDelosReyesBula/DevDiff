# Governance

DevDiff follows a community-driven governance model.

## Roles

| Role                                   | Responsibility                        | Selection              |
| -------------------------------------- | ------------------------------------- | ---------------------- |
| **BDFL**                               | Final decision authority (temporary)  | Project creator        |
| **TSC (Technical Steering Committee)** | Technical direction, RFC approval     | Elected by maintainers |
| **Maintainers**                        | Review PRs, triage issues, merge code | Nominated by TSC       |
| **Contributors**                       | Submit PRs, documentation             | Anyone!                |

## Decision Making

### RFC Process

Major changes (new providers, API changes, architecture) require an RFC:

1. Open GitHub Discussion with `RFC:` prefix
2. Community feedback period (minimum 2 weeks)
3. TSC vote (majority required)
4. Implementation

### Day-to-Day

- Bug fixes: any maintainer can merge
- Features: requires 1 maintainer review
- Breaking changes: requires 2 maintainer reviews + TSC discussion
